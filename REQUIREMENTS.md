# 视频时间轴编辑器 - 产品需求文档（PRD）

## 文档信息
- **产品名称**：视频时间轴编辑器
- **版本**：v1.2
- **创建日期**：2026年2月1日
- **文档类型**：产品需求文档

---

## 一、产品概述

### 1.1 产品定位
视频时间轴编辑器是一款基于Web的轻量级视频编辑工具，支持多轨道管理、智能排列、音乐卡点等功能，为用户提供简洁高效的视频编辑体验。

### 1.2 目标用户
- 视频创作者
- 短视频制作人员
- 需要快速编辑视频的普通用户

### 1.3 核心价值
- 直观的时间轴操作界面
- 智能的轨道管理系统
- 便捷的音乐卡点功能
- 自动保存和项目管理

## 二、功能需求

### 2.1 基础功能

#### 2.1.1 媒体导入与初始化
**需求描述**：用户可以从媒体选择器导入图片和视频，系统自动创建视频主轨。

**功能细节**：
- 支持多个媒体文件同时导入
- 视频自动获取真实时长（加载超时默认5秒）
- 图片默认时长3秒
- 媒体项按选择顺序在时间轴上首尾相连排列
- 初始状态视频主轨居中显示
- 第一个媒体项默认在视口中心位置

**时间轴规则**：
- 1秒 = 40像素
- 媒体项初始偏移量 = 视口中心位置

**交互流程**：
1. 用户在媒体选择器选择文件
2. 系统加载媒体元数据（视频时长）
3. 创建视频主轨，自动计算每个片段的位置
4. 默认显示第一个媒体项的预览

---

#### 2.1.2 轨道添加系统
**需求描述**：用户可以通过底部工具栏添加各类轨道到时间轴。

**轨道分类**：

| 类别 | 轨道类型 | 颜色代码 | 默认位置 | 默认时长 |
|------|---------|---------|---------|---------|
| 文字 | 文字 | #348CFF | 上方 | 3秒 |
| 文字 | 主轨字幕 | #21B7F1 | 上方 | 随机分割 |
| 文字 | 歌词字幕 | #0CD2D9 | 上方 | 随机分割 |
| 文字 | 画中画字幕 | #21B7F1 | 上方 | 随机分割 |
| 文字 | 录音字幕 | #21B7F1 | 上方 | 随机分割 |
| 音频 | 音乐 | #8E34FF | 下方* | 3秒 |
| 音频 | 音效 | #FF34C9 | 上方 | 3秒 |
| 音频 | 录音 | #FD2C75 | 上方 | 3秒 |
| 音频 | TTS | #FF34C9 | 上方 | 3秒 |
| 音频 | 提取音频 | #B71CFE | 上方 | 3秒 |
| 画面 | 贴纸 | #FF6247 | 上方 | 3秒 |
| 画面 | 特效 | #FF7F39 | 上方 | 3秒 |
| 画面 | 画中画 | 透明格子 | 上方 | 3秒 |

*注：在"1.2-主轨方案"下，音乐轨道默认添加到视频主轨下方；在"1.0-音+包"方案下添加到上方。

**智能行放置逻辑**：
1. 新轨道默认添加到视口中心位置
2. 检查同类型轨道（音频类/其他类）的现有行
3. 从第0行开始，检查是否有位置冲突
4. 如果有冲突，检查下一行
5. 如果所有行都有冲突，创建新行（行号+1）

**冲突判定规则**：
```
新轨道结束位置 > 现有轨道开始位置 
且 
新轨道开始位置 < 现有轨道结束位置
→ 判定为冲突
```

#### 2.1.3 字幕轨道随机分割
**需求描述**：添加字幕类轨道时，系统自动将其分割成多段，模拟真实字幕场景。

**字幕类轨道**：
- 主轨字幕
- 歌词字幕
- 画中画字幕
- 录音字幕

**分割规则**：
| 参数 | 取值范围 | 说明 |
|------|---------|------|
| 分段数量 | 2-5段 | 随机生成 |
| 第一段起始时间 | 1-5秒 | 随机，不超过视频总长度 |
| 每段时长 | 2-4秒 | 随机 |
| 段间间隔 | 1-3秒 | 随机 |

**生成流程**：
1. 计算视频主轨总时长
2. 随机确定分段数量（2-5）
3. 随机生成第一段起始时间（1-5秒内）
4. 循环生成每一段：
   - 随机时长2-4秒
   - 检查是否超出总时长
   - 计算下一段起始时间 = 当前段结束 + 随机间隔
5. 应用智能行放置逻辑，确保不重叠

### 2.2 轨道操作功能

#### 2.2.1 轨道选择
**需求描述**：用户可以通过点击选中轨道，为后续操作做准备。

**交互规则**：
- 单击轨道：选中（显示黄色边框 #FFC107）
- 双击轨道：显示操作气泡菜单
- 点击空白处：取消选中
- 同一时间只能选中一个轨道

**视觉反馈**：
- 选中状态：黄色边框（2px）+ 外发光效果
- 未选中状态：透明边框或默认颜色边框

---

#### 2.2.2 轨道拖动
**需求描述**：用户可以通过拖动调整轨道的时间位置和层级位置。

**水平拖动**：
- **触发条件**：选中轨道后，按住拖动
- **功能**：调整轨道在时间轴上的位置
- **限制**：不允许拖动到负数位置（左边界限制）
- **实时反馈**：拖动时轨道位置实时更新
- **例外**：视频主轨不支持拖动

**垂直拖动**：
- **触发条件**：选中附加轨道后，垂直方向拖动
- **功能**：在不同行之间移动轨道
- **行高**：60px为一行
- **拖动阈值**：垂直拖动距离达到60px时切换到下一行
- **冲突处理**：
  - 如果目标行有位置冲突，自动将目标行及之后的同类型轨道下移一行
  - 腾出空位供当前轨道插入

**拖动状态管理**：
- 拖动开始时记录初始位置（X、Y、行号）
- 拖动过程中锁定水平滚动位置（防止意外滚动）
- 拖动过程中**不锁定**垂直滚动（允许用户滚动查看）
- 拖动结束时标记轨道为"手动调整过"状态

---

#### 2.2.3 方向控制按钮
**需求描述**：选中轨道后，左右两侧显示精确位置调整按钮。

**按钮布局**：
```
[←] [↑]  [轨道]  [↓] [→]
 左侧组          右侧组
```

**按钮功能**：
| 按钮 | 功能 | 单次变化量 | 说明 |
|------|------|-----------|------|
| ← | 左移 | -40px（-1秒） | 不能移动到负数位置 |
| → | 右移 | +40px（+1秒） | 无限制 |
| ↑ | 上移一层 | rowIndex+1 | 检测冲突，自动腾空位 |
| ↓ | 下移一层 | rowIndex-1 | 检测冲突，不能低于0 |

**冲突处理**（上下移动）：
- **上移**：如果目标行有冲突，将目标行及之后的同类型轨道全部下移一行
- **下移**：如果目标行有冲突，不执行移动操作

**视觉样式**：
- 按钮大小：26×26px
- 背景色：紫色渐变 rgba(102, 126, 234, 0.9)
- 位置：距离轨道左右各60px
- 悬停效果：放大1.05倍
- 点击效果：缩小0.95倍

---

#### 2.2.4 轨道操作菜单
**需求描述**：双击轨道后显示操作气泡菜单，提供常用操作。

**菜单结构**：

**视频主轨菜单**：
- 复制

**附加轨道菜单**：
- 音乐卡点（仅音乐轨道显示）
- 变长（+3秒）
- 变短（-3秒，最短3秒）
- 复制
- 删除

**操作详情**：

1. **变长**：
   - 时长增加3秒
   - 如果是音乐轨道且已有卡点，自动在新增部分继续打卡点
   - 保持气泡打开，方便连续操作

2. **变短**：
   - 时长减少3秒
   - 最短不低于3秒
   - 保持气泡打开

3. **复制**：
   - 创建轨道副本
   - 副本偏移量 = 原轨道偏移 + 100px
   - 副本位置在原轨道上方（position: 'above'）
   - 副本标记为"手动调整过"
   - 复制后关闭气泡

4. **删除**：
   - 移除轨道
   - 清除相关的展开状态
   - 清除相关的卡点数据
   - 关闭气泡

**气泡样式**：
- 位置：轨道正上方40px
- 背景：黑色半透明 rgba(0, 0, 0, 0.9) + 模糊效果
- 带有向下的三角形指示器
- 按钮间距：0.3rem
- 按钮样式：深灰色背景 #2a2a2a，悬停变亮 #3a3a3a

### 2.3 音乐卡点功能

#### 2.3.1 卡点生成规则
**需求描述**：为音乐轨道自动生成节拍卡点，用于视频素材的节奏剪辑。

**卡点规律**：
```
1秒 → 1秒 → 1秒 → 3秒 → 循环
```

**生成算法**：
```javascript
intervals = [1, 1, 1, 3]
currentTime = 0
while (currentTime < 轨道总时长) {
  for (interval in intervals) {
    if (currentTime >= 轨道总时长) break
    创建卡点 at currentTime * 40px
    currentTime += interval
  }
}
```

**卡点属性**：
- 位置单位：像素（px）
- 转换关系：时间（秒）× 40 = 像素位置
- 视觉样式：黄色圆点 #FFD700
- 直径：8px（展开状态）/ 6px（折叠状态）
- 边框：1px 白色半透明
- 发光效果：0 0 4px rgba(255, 215, 0, 0.8)

**操作入口**：
1. 双击音乐轨道
2. 在操作气泡中点击"音乐卡点"按钮
3. 系统自动生成卡点

---

#### 2.3.2 卡点与轨道长度联动
**需求描述**：当音乐轨道增加长度时，自动在新增部分继续按规律打卡点。

**触发条件**：
- 音乐轨道已有卡点
- 用户点击"变长"按钮（+3秒）

**延续算法**：
```javascript
1. 找到最后一个卡点位置（转换为秒数）
2. 计算从最后一个卡点到旧时长结束的时间差
3. 确定当前在[1,1,1,3]循环中的位置
4. 如果距离最后卡点还没到下一个间隔，先补齐
5. 在新增的3秒内继续按规律打卡点
```

**示例**：
```
原轨道：6秒，已有卡点 [0s, 1s, 2s, 3s]
点击变长 → 9秒
新卡点：[0s, 1s, 2s, 3s, 6s, 7s, 8s]
         ↑ 原有     ↑ 新增（继续3秒间隔→1秒→1秒）
```

**数据存储**：
```javascript
musicBeatPoints = {
  trackId1: [0, 40, 80, 120, ...],  // 位置数组（px）
  trackId2: [0, 40, 80, 120, ...]
}
```

---

#### 2.3.3 卡点显示状态
**需求描述**：卡点在不同显示状态下自适应大小。

**显示规则**：
| 状态 | 音乐轨道高度 | 卡点直径 | 说明 |
|------|------------|---------|------|
| 展开状态 | 25px | 8px | 正常显示 |
| 折叠状态 | 3px | 6px | 吸底模式下自动折叠 |

**位置计算**：
```css
position: absolute
left: 卡点位置px
top: 50%
transform: translate(-50%, -50%)
```

**层级**：
- z-index: 10（确保在轨道内容之上）
- pointer-events: none（不阻挡轨道交互）

### 2.4 吸底与展开功能

#### 2.4.1 自动吸底机制
**需求描述**：当轨道过多超出容器高度时，自动将视频主轨吸附在底部，节省空间。

**触发条件**：
```
轨道总高度 > 容器可用高度
```

**高度计算**：
```javascript
轨道总高度 = 
  上方轨道行数 × 30px（每行）
  + 视频主轨 50px
  + 下方音乐轨道高度
```

**吸底实现方式**：
- 使用动态 `padding-top` 推动内容到底部
- `padding-top = 容器高度 - 内容高度`
- **不使用** flex 的 `justify-content: flex-end`（会影响滚动）

**吸底状态变化**：
1. **轨道较少**（默认）：
   - 视频主轨居中显示
   - 音乐轨道正常高度（25px）
   - 滚动容器居中对齐

2. **轨道过多**（吸底）：
   - 视频主轨固定在底部
   - 音乐轨道自动折叠（4px高）
   - 上方轨道可滚动查看

**视觉效果**：
- 视频主轨显示在固定的底部容器中
- 与滚动区域水平滚动同步
- 底部容器与上方区域有分割线（1px #333）

---

#### 2.4.2 向上滑动展开
**需求描述**：用户在吸底状态下向上滑动，系统自动恢复完全展开状态。

**触发条件**：
```
吸底状态 && 向上滚动距离 > 50px
```

**展开行为**：
1. 设置 `userManuallyExpanded = true`
2. 取消吸底状态 `shouldStickMainTrack = false`
3. 视频主轨回到滚动容器中
4. 所有音乐轨道恢复正常高度（25px）
5. 容器恢复顶部对齐方式

**状态持久性**：
- 用户手动展开后，即使增加新轨道也保持展开状态
- 切换方案时重置展开状态

---

#### 2.4.3 吸底状态下的交互
**需求描述**：确保吸底状态下所有轨道操作功能正常工作。

**必须保证的功能**：
1. ✅ 上方轨道可以选中
2. ✅ 上方轨道可以水平拖动
3. ✅ 上方轨道可以垂直拖动（换层）
4. ✅ 方向控制按钮正常工作
5. ✅ 双击显示操作菜单
6. ✅ 滚动容器可以上下滚动

**实现要点**：
- 拖动轨道时**不锁定**垂直滚动
- 滚动容器始终保持 `overflow-y: auto`
- 使用 `padding-top` 而非 flex 布局实现吸底

---

#### 2.4.4 双击音乐轨道切换状态
**需求描述**：吸底模式下，用户可双击音乐轨道手动展开/折叠单个轨道。

**功能规则**：
- 仅在吸底模式下生效
- 双击音乐轨道：展开（25px高度）
- 再次双击：恢复折叠（4px高度）
- 用户展开的轨道状态被记录

**状态管理**：
```javascript
userExpandedMusicTracks = Set([trackId1, trackId2, ...])
```

**显示逻辑**：
```javascript
if (shouldStickMainTrack && !userExpandedMusicTracks.has(trackId)) {
  // 显示折叠状态（3-4px高）
} else {
  // 显示展开状态（25px高）
}
```

### 2.5 方案切换系统

#### 2.5.1 方案概述
**需求描述**：提供两种预设编辑方案，适应不同的剪辑场景。

**方案对比**：

| 特性 | 1.0-音+包 | 1.2-主轨方案 |
|------|----------|------------|
| 音乐轨道位置 | 视频主轨上方 | 视频主轨下方 |
| 其他轨道位置 | 智能排列 | 视频主轨上方 |
| 吸底功能 | 不支持 | 支持 |
| 音乐折叠 | 不支持 | 吸底时自动折叠 |
| 适用场景 | 音乐+包装元素为主 | 以主视频为中心 |

---

#### 2.5.2 1.0-音+包方案
**布局规则**：
```
[其他轨道] ← 智能排列
[音频轨道] ← 智能排列
[视频主轨] ← 居中显示
```

**特点**：
- 所有附加轨道都在视频主轨上方
- 音频类和其他类分别管理行号
- 不启用吸底功能
- 适合音乐MV、包装类视频

---

#### 2.5.3 1.2-主轨方案
**布局规则**：
```
[其他轨道] ← 智能排列
[视频主轨] ← 核心位置
[音乐轨道] ← 默认位置，支持折叠
```

**特点**：
- 音乐轨道默认在主轨下方
- 支持自动吸底功能
- 音乐轨道可折叠显示
- 适合Vlog、纪录片等以主视频为中心的场景

---

#### 2.5.4 方案切换规则
**需求描述**：切换方案时清理状态，避免数据冲突。

**清理操作**：
1. 清空所有附加轨道
2. 清空卡点数据
3. 清空用户展开状态
4. 重置手动展开标志
5. 关闭所有菜单和气泡

**切换入口**：
- 点击顶部标题显示方案菜单
- 选择目标方案
- 确认切换

---

### 2.6 工程包系统

#### 2.6.1 自动保存
**需求描述**：编辑过程中自动保存项目状态，防止数据丢失。

**保存时机**：
- 防抖处理：编辑操作后1秒自动保存
- 避免频繁保存影响性能

**保存内容**：
```javascript
{
  projectName: "项目名称",
  currentScheme: "1.2-主轨方案",
  media: [...],                    // 视频主轨数据
  additionalTracks: [...],         // 附加轨道数据
  musicBeatPoints: {...},          // 卡点数据
  userExpandedMusicTracks: [...],  // 用户展开状态
  savedAt: "2026-02-01T12:00:00Z"  // 保存时间
}
```

**存储位置**：
- localStorage: `currentProject`
- 最大存储：10个项目

---

#### 2.6.2 手动导出
**需求描述**：用户可主动保存当前项目并命名。

**操作流程**：
1. 点击右上角"导出"按钮
2. 弹出项目名称输入框
3. 输入项目名称（必填）
4. 点击"确认保存"
5. 保存到 `currentProject` 和 `savedProjects` 列表
6. 显示保存成功提示

**命名规则**：
- 默认名称：`项目_YYYY-MM-DD`
- 用户可自定义修改
- 不允许空名称

---

#### 2.6.3 项目恢复
**需求描述**：从工程包列表恢复已保存的项目。

**恢复流程**：
1. 在媒体选择页面，用户点击工程包列表
2. 选择要恢复的项目

4. 从 localStorage 读取 `currentProject`
5. 恢复所有状态

**恢复内容**：
- ✅ 媒体文件列表
- ✅ 所有附加轨道位置和属性
- ✅ 音乐卡点数据
- ✅ 方案设置
- ✅ 用户展开状态
- ✅ 项目名称

#### 2.6.4 新项目与恢复项目区分

**判断逻辑**：
```javascript
if (location.state?.restoreProject) {
  // 从 localStorage 恢复项目
```

#### 卡点生成算法
```javascript
function generateBeatPoints(durationSec) {
  const intervals = [1, 1, 1, 3]
  const points = []
  let currentTime = 0
  
  while (currentTime < durationSec) {
    for (const interval of intervals) {
      if (currentTime >= durationSec) break
      points.push(currentTime * 40)
      currentTime += interval
    }
  }
  
  return points
}
```
